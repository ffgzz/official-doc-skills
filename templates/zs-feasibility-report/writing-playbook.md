# ZS-项目可行性报告写作 playbook

## 章节目标与建议写法

### 第1章 概述
- 目标：讲清楚“为什么要立项、想解决什么问题”
- 不要空泛喊口号，要把行业背景和立项意义对应起来

### 第2章 项目现状和发展趋势
- 目标：说明现有方法/产品的不足，以及行业下一步的发展方向
- 最好写清“现有流程为什么效率低、为什么口径不一致、为什么需要自动化/智能化”

### 第3章 研发内容及技术关键
- 目标：做任务分解
- 最好采用“总体研发内容 → 子任务 → 关键技术 → 创新点”的顺序
- 当参考样稿的三级标题不适用于新项目时，应按当前项目内容重新组织

### 第4章 技术路线和实施方案
- 目标：把项目从输入到交付的技术与实施链条讲清楚
- 这里必须为 `图1` 和 `表1` 留位

### 第5章 现有工作基础和条件
- 目标：回答“团队是否有能力做成这件事”
- 必须体现已有经验、既有成果、已有数据/设备/流程基础

### 第6章 困难评估
- 目标：不是罗列风险词，而是说明“难点是什么、为什么难、怎么应对”

### 第7章 预期目标
- 目标：拆成技术成果和应用成效两类，并尽量量化

### 第8章 计划进度安排
- 目标：阶段、时间、工作内容一一对应

### 第9章 经费预算
- 目标：说明预算结构、测算依据和口径
- 没有依据的分项金额不能乱写

### 第10章 研究结论
- 目标：从必要性、可行性、预期价值三方面做收束
