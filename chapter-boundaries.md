# Chapter Boundaries

本文件负责：定义整份公文中每一章、每一节分别写什么、不写什么。

流程、默认读取顺序、外部调研规则和通用约束以 `README.md` 与 `writing-rules/00-总流程与资料边界.md` 为准。

使用口径：

- 本文件用于跨章节去重和统一口径，不替代 `chapter-prompts/` 的结构骨架，也不替代 `writing-rules/` 的写法要求。
- 本文件提供全书级边界概览；具体落到某一章时，以该章的 `chapter-prompts/` 和 `writing-rules/` 中的细化口径为准。
- 同一事实尽量只在一个“主归属章节”完整展开；其他章节如确需提及，只能做简短承接，不得整段重复。
- 同一术语、系统名称、平台名称、单位简称、成果名称、指标口径，一旦依据 `reference-materials/<project-slug>/` 确定，后文必须保持一致。

## 内容主归属快表

| 内容主题 | 主归属章节 | 其他章节可提及方式 |
|----------|------------|--------------------|
| 政策背景、行业变革、国内外趋势 | 第一章 | 其他章只承接，不重复铺陈 |
| 单位简介、单位角色、负责人 | 第二章 | 第六章可按分工再引用 |
| 团队基础、既往业绩、软硬件条件 | 第三章 | 第七章可简要承接保障条件 |
| 项目总体目标、问题、研发内容、技术路线、预期成果、推广路径、集成方式、成果受控复用原则 | 第四章 | 第五至第七章只能按各自职责局部承接 |
| 任务拆解、阶段安排、里程碑 | 第五章 | 第四章只讲“建设内容”，不写正式任务管理口径 |
| 联合体成员分工、交付边界 | 第六章 | 第四章第七节只讲协同研发与集成逻辑 |
| 组织管理、运行保障、成果应用管理、权益分配、应用推广迭代治理 | 第七章 | 第四章只讲方案视角，不讲正式治理制度 |
| 投资估算、资金来源、资金管控 | 第八章 | 第九章只引用必要口径，不重写预算明细 |
| 财务效益、经济效益、社会效益 | 第九章 | 第一章和第四章只能写高层价值，不做正式测算 |
| 风险识别与对策 | 第十章 | 其他章节只可点到，不做系统风险清单 |

## 第一章 边界

### `一、项目背景及必要性`

- 本章主任务：回答“为什么现在要做、为什么值得做、为什么必须做”。
- 本章可写：政策背景、行业背景、业务场景背景、国内外现状、公开趋势、项目拟解决的高层问题、对产业链供应链韧性及安全的意义。
- 本章不写：单位详细能力、团队详细业绩、正式任务分解、正式技术路线、成果管理制度、预算测算、正式收益测算、正式风险清单。

#### `（一）建设背景`

- 只写“宏观政策 -> 行业背景 -> 业务场景 -> 当前时机”。
- 不提前写建设意义、正式问题清单、预算、成果清单。

#### `（二）建设意义`

- 只写“技术层面、产业层面、生态层面”的意义。
- 不写正式成果明细、单位责任分配、具体推广对象。

#### `（三）国内外发展现状及前景`

- 只写公开世界里的现状、短板、对比和趋势。
- 不把外部案例写成本项目既有基础。

#### `（四）预期解决重大问题`

- 只写必要性视角的高层问题链，即“为什么这些问题必须被项目处理”。
- 不替代第四章 `2. 项目解决的主要问题` 的工程化问题拆解。

#### `（五）对产业链供应链韧性及安全的意义`

- 只写价值和意义，不写正式成果复用制度、推广制度、审批制度。

## 第二章 边界

### `二、项目单位基本情况`

- 本章主任务：回答“谁来做、为什么这些单位/负责人具备承担资格”。
- 本章可写：牵头单位、应用验证单位、研发承担单位、负责人、其他支撑单位的正式背景与角色定位。
- 本章不写：团队详细过往项目列表、正式任务分工、正式管理制度、预算安排。

#### `（一）项目申报主体及牵头单位情况`

- 只写申报主体和牵头单位的性质、定位、能力边界。
- 不展开到具体研发成果和项目履历长清单。

#### `（二）需求提出及应用验证单位情况`

- 只写需求来源和验证场景主体。
- 不写详细试点计划和推广路径。

#### `（三）研发承担单位情况`

- 只写研发主体情况和承担理由。
- 不替代第六章的正式分工表。

#### `（四）项目负责人及主要团队情况`

- 只写负责人和核心团队构成。
- 团队详细业绩、软硬件条件主归第三章。

#### `（五）其他支撑单位或联合体成员情况`

- 只写支撑关系与参与必要性。
- 不写接口流程、验收边界和成果分配。

## 第三章 边界

### `三、项目团队工作基础`

- 本章主任务：回答“团队过去做过什么、现在有什么基础、凭什么能落地”。
- 本章可写：团队情况、团队实力、软硬件条件、既往业绩、专业资质能力。
- 本章不写：单位简介重复、正式任务分工、正式管理制度、正式预算和收益测算。

#### `（一）团队情况`

- 只写团队构成和基本情况。
- 不重复第二章的单位简介。

#### `（二）团队实力和基础`

- 只写能力基础和技术积累。
- 不写完整项目建设方案。

#### `（三）软硬件支撑条件`

- 只写已有条件和能力边界。
- 不写未来采购计划和投资估算。

#### `（四）以往业绩及承担相关项目情况`

- 只写可支撑本项目的既往工作。
- 不把历史项目写成当前项目的正式成果。

#### `（五）专业人员资质能力情况`

- 只写专业资质、能力和匹配度。
- 不展开到第六章式分工。

## 第四章 边界

### `四、项目建设方案`

- 本章主任务：回答“项目具体要建什么、解决什么、怎么建、产出什么、如何推广和协同、成果如何原则性管理”。
- 本章可写：目标、问题、研发内容、技术路线、成果、推广路径、集成方式、成果受控复用原则。
- 本章不写：正式任务管理台账、正式单位分工表替代、第七章式治理制度、第八章预算、第九章正式效益测算、第十章正式风险清单。

#### `（一）总体目标`

##### `1. 建设目标`

- 只写项目完成后应达到的能力状态、应用对象、验证边界和总体水平。
- 不展开到详细技术路线、详细任务拆分、详细单位分工。

##### `2. 项目解决的主要问题`

- 只写建设方案视角的工程问题、系统问题、实现问题。
- 不重复第一章的宏观痛点背景。
- 不直接展开成正式研发任务。

##### `3. 项目研发内容`

- 只写要研发什么、各子课题/各任务块负责什么技术内容。
- 不写正式阶段进度表和单位责任分配。

##### `4. 预期成果`

- 只写项目计划形成的成果类别、成果价值、成果口径。
- 表格只允许落资料已支撑的成果类别、数量和保守边界。
- 不自行补具体专利名称、软著名称、成员分配。

##### `5. 产业链供应链韧性及安全保障`

- 只写项目成果对自主可控、数据沉淀、平台安全、协同韧性的支撑意义。
- 不补写具体制度、具体权限、具体审批链。

#### `（二）项目建设方案`

##### `1. 技术路线`

- 只写技术路线、数据流、业务流、模块接口、测试验证和工程应用路径。
- 不替代第五章的正式任务计划。

##### `2. 应用推广方案`

- 只写方案视角下的试点验证、内部推广、外部复制前提、长期迭代原则。
- 不写第七章式正式推广治理机制。
- 资料不足时，外部复制和运维只能保守表述。

##### `3. 联合研发与平台集成方式`

- 只写联合研发模式、平台集成关系、协同研发和联调逻辑。
- 不替代第六章的正式分工与交付边界。

##### `4. 成果管理与内部受控复用策略`

- 只写成果分类、受控复用边界、知识产权与保密、对外发布边界、原则性闭环。
- 不替代第七章的正式权益分配与治理制度。
- 不自行补写具体审批部门、协议文本、备案层级。

## 第五章 边界

### `五、项目任务设置`

- 本章主任务：回答“项目正式怎么拆任务、分阶段怎么推进、里程碑是什么”。
- 本章可写：任务总体划分、分任务展开、任务协同与阶段安排。
- 本章不写：技术路线细节、单位正式职责、组织管理制度、预算测算。

#### `（一）任务总体划分`

- 只写任务结构。
- 不重复第四章高层目标与意义。

#### `（二）分任务展开`

- 只写任务条目、输入输出、交付物。
- 不替代第六章单位分工。

#### `（三）任务协同与阶段安排`

- 只写任务衔接和里程碑。
- 不展开成第七章治理制度。

## 第六章 边界

### `六、联合体成员单位任务分工情况`

- 本章主任务：回答“哪个单位具体承担什么、交什么、接口边界在哪里”。
- 本章可写：总体分工说明、牵头单位职责、参与单位职责、协同接口与交付边界。
- 本章不写：第四章式技术路线、第五章式任务管理逻辑、第七章式治理制度。

#### `（一）分工总体说明`

- 只写分工原则和总体格局。

#### `（二）牵头单位职责`

- 只写牵头单位承担内容。
- 不扩成单位简介。

#### `（三）参与单位职责`

- 只写参研单位各自职责。
- 不展开行业背景和既往成绩。

#### `（四）协同接口与交付边界`

- 只写接口关系、交付边界、联调方式。
- 不写审批制度和权益分配。

## 第七章 边界

### `七、项目组织及实施管理`

- 本章主任务：回答“项目建成与实施过程中如何组织、保障、治理、应用和迭代”。
- 本章可写：管理模式、运行保障、人才与设备保障、成果应用管理、知识产权及权益分配、成果应用推广与迭代策略。
- 本章不写：第一章式背景意义、第四章式技术路线、第五章式正式任务清单、第八章式预算测算、第九章式收益测算。

#### `（一）项目管理模式`

- 只写组织管理架构和职责运转方式。

#### `（二）项目运行保障机制`

- 只写实施保障、质量保障、数据/保密/运维保障。
- 不替代第十章风险清单。

#### `（三）项目人才团队及设备设施保障`

- 只写实施期保障，不重复第三章历史基础。

#### `（四）项目成果应用管理`

- 只写成果应用管理原则和组织安排。
- 不替代第四章 `4. 成果管理与内部受控复用策略` 的方案视角表达。

#### `（五）知识产权及权益分配`

- 只写治理和权益安排。
- 不重复第四章成果类别本身。

#### `（六）项目成果应用推广与迭代策略`

- 只写实施治理视角的推广与迭代机制。
- 不重复第四章 `2. 应用推广方案` 的试点路径图和场景范围图。

## 第八章 边界

### `八、资金筹措及投资估算`

- 本章主任务：回答“项目需要多少钱、钱怎么来、怎么用、怎么管”。
- 本章可写：投资估算依据与原则、总投资估算、资金筹措方案、资金使用与管控要求。
- 本章不写：正式收益测算、市场前景大段背景、技术路线细节。

#### `（一）投资估算依据与原则`

- 只写估算口径和边界。

#### `（二）项目总投资估算`

- 只写投资构成和总量。
- 不展开资金来源逻辑。

#### `（三）资金筹措方案`

- 只写资金来源和配套关系。
- 不写收益测算。

#### `（四）资金使用与管控要求`

- 只写管控原则和边界。
- 不自行补内部审批制度。

## 第九章 边界

### `九、财务经济效益测算`

- 本章主任务：回答“项目做成后在经济和社会层面带来什么效益，以及这些效益如何测算或判断”。
- 本章可写：经济效益分析、社会效益分析。
- 本章不写：预算来源、投资估算明细、背景政策大段铺陈、正式风险清单。

#### `（一）经济效益分析`

- 只写收益机制、降本增效逻辑、测算边界。
- 不重复第八章预算表。

#### `（二）社会效益分析`

- 只写安全、行业带动、标准化、示范作用等社会价值。
- 不替代第一章高层意义和第四章成果说明。

## 第十章 边界

### `十、项目综合风险因素分析`

- 本章主任务：回答“项目可能面临哪些风险，各自如何防范”。
- 本章可写：技术、市场、经营、资金、法律、政策风险及对策。
- 本章不写：再讲一遍项目背景、成果清单、预算明细、推广路径。

#### `（一）技术风险评价与防范对策`

- 只写技术层风险与对应防范。

#### `（二）市场风险评价与防范对策`

- 只写市场接受度、推广条件、竞争环境等风险。

#### `（三）经营风险评价与防范对策`

- 只写组织运营、实施协调、资源配置等风险。

#### `（四）资金风险评价与防范对策`

- 只写资金到位、成本波动、投资控制等风险。
- 不重复第八章估算明细。

#### `（五）法律风险评价与防范对策`

- 只写合规、知识产权、保密、合同边界等风险。
- 不替代第七章和第四章的治理边界说明。

#### `（六）政策风险评价与防范对策`

- 只写政策变化、监管要求变化和应对。

## 高重复风险对照

- 第一章 `（四）预期解决重大问题` 与 第四章 `2. 项目解决的主要问题`
  - 第一章写“为什么必须解决”
  - 第四章写“建设方案层面具体要解决什么”

- 第四章 `3. 项目研发内容` 与 第五章 `项目任务设置`
  - 第四章写“技术建设内容”
  - 第五章写“正式任务拆解、阶段安排、里程碑”

- 第四章 `3. 联合研发与平台集成方式` 与 第六章 `联合体成员单位任务分工情况`
  - 第四章写“协同研发逻辑与平台集成关系”
  - 第六章写“单位责任、交付边界、接口边界”

- 第四章 `2. 应用推广方案` 与 第七章 `项目成果应用推广与迭代策略`
  - 第四章写“方案路径和场景扩展”
  - 第七章写“组织治理、推广管理和迭代机制”

- 第四章 `4. 成果管理与内部受控复用策略` 与 第七章 `知识产权及权益分配`
  - 第四章写“成果分类、复用边界、原则性管理”
  - 第七章写“治理安排、权益分配、管理责任”

- 第八章 `资金筹措及投资估算` 与 第九章 `财务经济效益测算`
  - 第八章写“花多少钱、钱怎么来”
  - 第九章写“做成后带来什么效益”
