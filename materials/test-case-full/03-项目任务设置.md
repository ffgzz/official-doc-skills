# 项目任务设置

## 一、项目周期

- 建设开始时间：`2024年5月`
- 建设结束时间：`2026年4月`
- 总周期：`24个月`

## 二、任务总体框架

### 任务一：软件基础平台顶层设计

- 开展平台框架分级流程分析
- 完成业务架构、数据架构、应用架构、技术架构 `4A` 分析
- 制定标准体系、测试环境方案和开源体系建设方案

### 任务二：业务架构应用关键技术

- 基于模型的船舶总体全生命周期管理应用场景分析
- 船舶厂所一体化协同数据包发放与接收管理应用研究
- 船舶总体跨平台协同意见跟踪管理技术研究
- 船舶配套装备研发设计全过程应用研究

### 任务三：数据架构关键技术

- 基于元数据的船舶全生命周期全要素信息定义
- 面向全生命周期的通用信息模型管理
- 面向全过程的船舶数据生态系统构建
- 船舶仿真数据管理技术

### 任务四：技术开发架构关键技术

- 基于知识图谱的产品数据建模
- 基于服务网格的复杂业务服务编排
- 基于微前端的低代码界面开发
- 面向船舶产品的三维模型轻量化及可视化
- 船舶总体全过程多模态数据管理

### 任务五：核心业务功能研发

- 需求管理模块
- 工程项目管理模块
- 设计规划管理模块
- 产品数据管理模块
- 设计协同管理模块
- 质量过程管理模块
- 知识库管理模块
- 系统管理、系统配置、元数据管理、技术状态管理、全文检索等服务

### 任务六：集成接口与数据输出功能

- 完成需求模型、总体设计、仿真、计划、制造、采购等数据集成接口建设
- 形成统一输出和数据流通能力
- 支撑跨软件、跨单位的数据贯通

### 任务七：测试验证与工程应用

- 开展核心功能测试
- 开展专业测试机构测试
- 开展信创环境适配测试
- 开展典型船舶、海工与船用配套设备工程应用

## 三、任务与成果对应关系

| 任务 | 主要交付 |
|---|---|
| 任务一 | 顶层设计方案、标准体系方案、测试环境方案、开源体系方案 |
| 任务二 | 业务架构应用研究成果、协同应用方案 |
| 任务三 | 数据模型与数据管理关键技术方案 |
| 任务四 | 技术开发架构与核心算法方案 |
| 任务五 | 平台核心业务功能模块 |
| 任务六 | 多类数据集成接口与输出能力 |
| 任务七 | 测试报告、工程应用结果、验收支撑材料 |

## 四、关键节点安排

### 第一年度：2024.05 - 2025.04

- 完成平台顶层设计
- 完成数据标准调研和标准起草
- 完成测试环境方案设计
- 完成业务架构、数据架构、技术架构关键技术方案论证
- 完成核心业务功能建设和内部测试
- 完成主要接口建设
- 完成项目中期报告

### 第二年度：2025.05 - 2026.04

- 完成核心功能测试
- 完成信创环境搭建和国产化适配测试
- 完成软件集成接口测试
- 完成多类典型产品工程应用
- 完成单项验收、综合验收和项目总结
