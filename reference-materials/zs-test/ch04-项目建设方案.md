## 01 总体目标可用材料

- 项目总体目标是开发“船体结构疲劳分析报告自动生成”系统。
- 目标链路是：从有限元计算结果提取开始，经机器学习分类和有限元自动化脚本处理，最终输出满足船级社要求的合规报告。
- 源文档强调要深度融合有限元分析技术、机器学习分类技术与大语言模型。
- 项目被定义为端到端自动化工作流程，意图打通“计算结果解析 -> 节点识别 -> 应力云图输出 -> 模板绑定 -> 报告生成”。

## 02 项目解决的主要问题可用材料

- 人工流程耗时高：单个 DA 大区约 100 工时，全船约 400 至 500 工时。
- 人工转录与编制易错：退审主因之一是云图标注不规范或计算结果格式偏差。
- 数据沉淀不足：点云数据与损伤计算结果没有结构化复用。
- 船体疲劳节点几何参数复杂多样，参数化建模难度高。
- 监督学习、无监督学习和半监督学习协同效率不高，标签传播可能引入噪声样本。
- 视角定位在均匀网格或对称结构下不稳定，影响云图输出质量。
- 动态模板引擎在处理不同节点类型、应力水平、规范版本时分支复杂，维护困难。

## 03 项目研发内容可用材料

### 子课题1：基于机器学习算法的船体有限元模型特征抓取

- 子课题1.1 数据集生成的自动化方法
  - 识别三类常见疲劳敏感节点：腹板加筋型十字节点、自由边型节点、焊接趾端型节点。
  - 对节点进行参数化定义，覆盖板厚、角度、圆角半径、加劲肋长度等参数和拓扑约束。
  - 通过 CAD 系统或脚本实现自动化建模、特征面划分、自动网格划分和统一格式导出。
- 子课题1.2 数据预处理与特征工程
  - 支持 bdf/dat/nas 等多源异构数据统一转换。
  - 建立 KD-Tree 或 Octree 空间索引，进行节点查重与合并。
  - 开展网格质量诊断，提取几何、拓扑、力学多维特征。
- 子课题1.3 混合式节点分类模型
  - 构建几何特征、拓扑特征、力学特征统一描述体系。
  - 采用“区域初筛 + 节点精细分类”的分层分类架构。
  - 结合随机森林、SVM、MLP、GNN 以及 K-means、DBSCAN、GraphSAGE、Semi-GCN 等方法。
- 子课题1.4 节点分类结果可视化
  - 将分类结果映射回三维网格模型。
  - 对不同类型节点进行着色高亮，并展示节点 ID、单元 ID、分类标签。

### 子课题2：有限元软件的自动化脚本开发

- 面向目标节点自动输出应力分布图像，并具备批量处理能力。
- 子课题2.1 节点智能定位与视角优化
  - 基于分类模型返回的节点信息定位目标单元。
  - 通过 PCA 计算最优观察视角。
  - 通过包围盒对角线长度自适应调整视距。
- 子课题2.2 应力分布云图输出
  - 解析 op2 文件并读取应力计算数据。
  - 自动生成主应力分布云图并保存为图像文件。
- 子课题2.3 模型批量化处理
  - 自动定位目标节点
  - 自动展示优化视角
  - 自动生成应力云图并输出结果

### 子课题3：基于大语言模型的报告生成研究

- 子课题3.1 规范知识图谱构建
  - 将船舶行业规范、疲劳分析指南和历史报告转成结构化知识。
  - 通过语义分割和标签化进行数据结构化。
  - 建立规则引擎和数值校验机制。
- 子课题3.2 动态模板引擎
  - 设计带目录索引、条件分支和动态表格的结构化模板。
  - 自动将分析结果、应力云图等绑定到模板占位位置。
- 子课题3.3 智能体构建
  - 利用大语言模型自动生成报告初稿。
  - 利用语义修正与单位校验机制保障报告规范性。
  - 支持 Word/PDF 输出、图表插入、编号和交叉引用。
