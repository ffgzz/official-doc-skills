# ch04-05-技术路线

## 本节目标

本节是第四章最关键的小节，要说明项目准备按什么架构和链路实现前面提出的任务与目标。

## 论证顺序

固定建议采用：

`总体技术路线总述 -> 架构分层 -> 数据流/业务流 -> 模型链/工具链 -> 模块和接口关系 -> 验证路线 -> 工程应用路线 -> 安全与自主可控`

## 必须覆盖的内容

- 总体架构
- 数据流和业务流
- 任务之间的衔接关系
- 模块/接口关系
- 测试验证安排
- 工程应用路径

## 句式与文风要求

- 先总后分
- 先讲架构，再讲链路，再讲验证
- 不要只写“采用人工智能、大数据等技术”

## 禁止事项

- 不要把技术路线写成技术名词清单
- 不要只写一个概念图说明
- 不要和研发内容完全重复

## 深度要求

- 本节通常应是第四章最长的小节之一
- 至少写清 4 层以上结构关系或等价链路
- 最好能自然引出图表或对应表
