# ch01-项目背景及必要性

## 本章目标

本章要完成三件事：

1. 说明项目为什么现在必须做
2. 说明项目做了以后有什么技术、产业和生态意义
3. 说明国内外现状、核心痛点和未来前景，并自然引出后续建设方案

## 论证顺序

固定采用：

- `建设背景`
- `建设意义`
- `国内外发展现状及前景`
- `预期解决重大问题`
- `对产业链供应链韧性及安全的意义`

其中重点链路必须形成：

`政策/行业背景 -> 现实短板 -> 项目切入 -> 建设意义 -> 国内外对比 -> 痛点 -> 前景 -> 重大问题 -> 安全与韧性`

## 句式与文风要求

- 先写行业与场景，再落到本项目
- 先写现状，再写问题，再写项目切入
- 每个“意义”都要写成“动作 -> 形成能力 -> 作用对象 -> 影响”
- 国内外现状不要写成资料堆砌或百科条目

## 必须覆盖的内容

- 背景中的政策、行业、技术和业务场景
- 建设意义中的技术、产业、生态三层
- 国内现状、国外现状、痛点分析、发展前景
- 本项目预期解决的重大问题
- 对产业链供应链韧性及安全的具体作用

## 禁止事项

- 不要只写“意义重大”
- 不要把国内外现状写成“国外先进、国内不足”
- 不要把痛点写成抽象抱怨
- 不要把发展前景写成空泛展望
- 不要新增模板未登记的小节

## 深度要求

- 痛点至少形成 3 到 4 类问题链
- 每类痛点都要能回指后续任务或技术路线
- 重大问题要与项目实际方向对应，不能泛化

## 可参考写法

- `面对……现实压力和……建设需求，亟须围绕……开展系统研究和工程化建设。`
- `在技术层面，项目将通过……建立……能力，支撑……场景下的……。`
