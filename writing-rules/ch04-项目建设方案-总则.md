# ch04-项目建设方案-总则

## 本章目标

第四章必须建立完整闭环：

`目标 -> 问题 -> 任务 -> 技术路线 -> 输出 -> 验证 -> 推广 -> 协同/管理边界`

## 结构要求

固定采用：

- `（一）总体目标`
- `（二）项目建设方案`

`01-04` 对应前者，`05-08` 对应后者。

## 统一文风要求

- 这是整份文稿中最工程化、最具体的一章
- 不能写成“方案摘要”
- 不能只写任务名和技术名词
- 每个子章都要和前后子章形成承接

## 统一深度要求

- 问题必须能回指第一章痛点
- 任务必须能回指问题
- 技术路线必须能回指任务
- 成果必须能回指任务和验收
- 推广、协同和成果管理必须能回指成果形态和应用对象

## 禁止事项

- 不要重复输出不属于当前子文件的上级标题
- 不要把研发内容和技术路线写成同一件事
- 不要让目标、任务、成果和验收脱节
