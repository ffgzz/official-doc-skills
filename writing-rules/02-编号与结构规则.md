# 02-编号与结构规则

## 默认编号方案

正式报告型文稿默认采用：

- 一级章：`一、`
- 二级节：`（一）`
- 三级项：`1.`
- 四级点：`（1）`

## 图表编号方案

- 插图统一使用 `图X-Y` 编号。
- 表格统一使用 `表X-Y` 编号。
- `X` 必须对应一级章号，`Y` 必须对应该章内部的连续序号。
- 同一一级章内，图和表分别连续编号，不能混号。
- 第四章虽然拆成多个子文件，但图表编号仍按整章连续编号，不因子文件重置。
- 图表标题必须与正文引用口径一致，不能正文写“如下图所示”而图题与内容脱节。
- 图的正文引用默认采用 `……如图X-Y所示。`
- 表的正文引用默认采用 `……见表X-Y。`
- 表标题默认采用 `**表X-Y 表名**` 单独成行

## 硬规则

- 同一层级不得混用 `（1）` 和 `1.1`
- 不得把正文写成连续编号清单
- 标题下必须有完整自然段展开
- 不得把一级标题降成普通小标题使用
- 图表前后必须有引出和解释文字，不能只放图题或表题
- 未被chapter-prompts要求、且资料无法支撑的图表不得为了凑版式硬加
- 条件性图表只有在资料足够支撑时才启用；若不启用，不得在正式文稿中留下提示语
- Mermaid 代码块必须使用 fenced code block，info string 固定为 `mermaid`

## 结构边界

- 只允许在模板给出的结构里展开
- 不得新增模板未登记的二级节
- 字数不足时，应在已有小节内补充事实、机制、输出、验证
- 不得为了凑字数新增“展望、意义、建议、历史价值”等计划外标题

## 第四章特殊规则

第四章 `项目建设方案` 的二级结构固定为：

- `（一）总体目标`
- `（二）项目建设方案`

第四章子文件必须服从：

- `01-04` 属于 `（一）总体目标`
- `05-08` 属于 `（二）项目建设方案`

子文件不得重复输出不属于自己的上级标题。
