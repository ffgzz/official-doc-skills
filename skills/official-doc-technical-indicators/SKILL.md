# 主要技术指标 Skill

## 先做什么

开始写之前，必须搜索：
- 同类项目指南、任务书、建议书中的指标写法
- 相关标准、规范、测试方法
- 行业基准、论文实验结果、官方案例

同时必须读取：
- 研究内容
- 技术成果

## 基本原则

技术指标必须满足四个条件：

1. 可量化
2. 可测试
3. 可回指研究内容
4. 可回指技术成果

## 推荐指标类型

常见主指标可从以下几类选择：
- 准确率、识别率、召回率、覆盖率
- 设计效率提升、工时缩短、自动化率
- 并发数、响应时间、处理规模
- 数据集规模、规则条目数、样本量
- 应用验证数量、场景覆盖数

是否加入产业化、经济性、知识产权指标，取决于用户章节要求。

## 写法要求

每个指标至少写清：
- 指标对象
- 指标名称
- 阈值或目标值
- 单位
- 说明或测试口径

优先写成：
- ……准确率 ≥ 90%
- ……效率提升 ≥ 40%
- ……并发用户数不少于 50
- ……样本规模不少于 2000 条

## 阈值处理

阈值来源必须有依据。

若用户未给明确阈值：
- 优先参考公开同类项目与标准
- 只在依据较强时写建议阈值
- 依据不足时写 `【待确认】`

不要为了让表好看而硬填数字。

## 常见结构

推荐先总述，再分组：
- 平台能力指标
- 核心算法指标
- 工程应用指标
- 支撑资源指标

如果用户要求简洁，可直接列主指标清单。

## 与成果的关系

写完后应能回答：
- 哪个研究内容支撑该指标
- 哪项成果承载该指标
- 后续如何验收该指标

## 输出检查

写完后确认：
- 是否每条都有单位/阈值/说明
- 是否存在明显不可测指标
- 是否有夸大或无依据数值

## 不要做的事

- 不要写“达到国际领先水平”这类不可测指标
- 不要把愿景口号当指标
- 不要让指标脱离研究内容和成果
